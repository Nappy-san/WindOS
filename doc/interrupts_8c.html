Detailed Description

Interrupt handling features.

This code is called by interrupts_asm.S to handle ARM interupts.

Function Documentation

void data_abort_vector ( void *  data)

Data abort interrupt handler.

When the MMU signals a data abort, check if it caused by the kernel or a process. In case of a process, kill it. If this is the kernel, branch into the last resort debug tool.

void interrupt_vector ( void *  user_context)

Hardware interrupt handler.

Parameters
void*Pointer to the usermode context.

The usermode context is gathered by interrupts_asm.S It should be saved on the current process data.

If the interruption is triggered by the Timer, it performs a process switch. The scheduler choses a new process and updates user_context to this new process' context. When this function returns, the context is restored.
